download

PROTOCOL

This is what I am going to call "Simple Chess Protocol". The communication use simple text files. The default names are 'input.txt' for engine to GUI and 'output.txt' for GUI to engine, but every engine should be able to use other names. Moves are passed in long algebraic notation. For example a2a3, d2d3. Promotion is doing by adding a 'q', 'r', 'n' or 'b' after the pawn move (not implemented in the GUI yet which shows buttons for you to choose). If not present, the 'q' will be de default. Only the first line of the text files are used. Every time the engine erase the file and put the move or the command in the first line. The casteling is done by e1h1, e8h8, etc.

Commands from GUI to Engine.
-go (make the engine to play for the side to move.
-goall (engine plays all moves).
-new (start a new game).
-quit (the engine must exit).

Commands from Engine to GUI.
-1-0 (White has won the game).
-0-1 (Black has won the game).
-1/2-1/2 (Stalemate).
-draw (the game is a draw by any other reason than stalemate, for example 50 moves rule, 3 fold repetion, etc. No support for agreed draw).

Time.
All engines must have one or more configuration files. All time conditions are going to be set in these files. But this is just a recommendation. There is not going to have loses because time.

Two different engines at same time.
To make one engine to play with another you don't even need the GUI! :) Just set the output and input of one engine to the input and output of the other. The GUI, if present, must be in read only mode (not implemented yet). In the configuration file(s) must be set if one engine is going to play white or black

Opening book.
Opening books are left for the engines. I do not recommend them. I think no opening book is better.
